# NomadReady — Implementation Rules

## General Philosophy

Implement interfaces as editorial scenes, not dashboards.

Every section should feel composed, atmospheric, and intentional.

The experience should feel like:
- a premium travel artifact
- a cinematic field guide
- a tactile editorial publication

Avoid generic startup UI patterns.

---

# Preferred Stack

Preferred technologies:
- Next.js
- Tailwind CSS
- SVG overlays
- CSS gradients
- lightweight motion
- compressed MP4/WebM atmospheric loops

Avoid:
- heavy WebGL
- unnecessary Three.js systems
- complex GPU-heavy scenes
- overengineered animations

---

# Layout Rules

Desktop layouts must:
- breathe
- use negative space
- feel cinematic
- create pacing
- support atmosphere

Prefer:
- asymmetry
- layered sections
- immersive visual panels
- offset compositions

Avoid:
- overly centered layouts
- dense dashboard spacing
- equal spacing everywhere
- tiny cards floating in whitespace

---

# Hero Rules

Hero sections should:
- feel immersive
- preserve readability
- contain atmospheric depth
- support cinematic layering

Hero imagery must include:
- negative space for text
- restrained lighting
- calm visual zones

Avoid:
- cluttered hero compositions
- aggressive motion
- loud gradients
- startup-style CTAs

---

# Passport Gateway Rules

The passport object should:
- feel tactile
- use warm shadows
- contain embossed details
- use restrained 2.5D movement

Avoid:
- glossy reflections
- gaming aesthetics
- sci-fi visuals
- excessive motion

Hover behavior:
- subtle lift only
- gentle perspective tilt
- cinematic restraint

---

# Featured Destination Panels

Featured panels are large editorial spreads.

They are not cards.

Each panel should:
- occupy large desktop space
- contain cinematic imagery
- preserve breathing room
- feel immersive
- support layered atmospheric depth

Recommended structure:
- cinematic image
- editorial copy
- restrained travel data
- subtle travel artifacts

Avoid:
- dashboard layouts
- repetitive card grids
- excessive UI chrome

---

# Destination Cards

Destination cards remain secondary.

Purpose:
- scanning
- navigation
- discovery

Cards should:
- feel tactile
- use warm surfaces
- contain subtle depth
- remain visually restrained

Avoid:
- fintech card styling
- widget aesthetics
- cold UI patterns

---

# Editorial Assets

Editorial assets include:
- visa stamps
- route tickets
- postcards
- atlas marks
- field notes
- compass overlays

Rules:
- low opacity
- subtle layering
- warm ink tones
- integrated composition

Do not use them as decorative stickers.

Preferred behavior:
- partially hidden
- softly layered
- atmospheric

---

# Atlas / Background Rules

Maps should behave as:
- atmosphere
- depth
- environmental texture

Never:
- dashboard widgets
- navigation systems
- interactive Google Maps clones

Preferred effects:
- embossed contours
- faded routes
- subtle cartographic texture
- atmospheric gradients

---

# Shadows

All shadows must remain warm.

Preferred tones:
- warm black
- amber diffusion
- parchment depth

Avoid:
- cool gray shadows
- harsh black shadows
- glowing neon effects

Hover shadows should feel soft and physical.

---

# Motion Rules

Motion must feel environmental.

Allowed:
- subtle fades
- atmospheric movement
- restrained hover lift
- soft transitions
- cinematic pacing

Avoid:
- bouncing
- playful motion
- exaggerated scaling
- flashy transitions
- startup microinteractions

Preferred durations:
- 150–260ms for state changes
- slower cinematic fades only when necessary

---

# Video Rules

Video should only support atmosphere.

Recommended usage:
- Tokyo rain
- drifting steam
- lantern movement
- tropical haze
- subtle reflections

Video should:
- loop seamlessly
- remain slow
- stay visually restrained

Avoid:
- cinematic trailers
- aggressive motion
- autoplay-heavy experiences
- travel vlog aesthetics

---

# Typography Rules

Typography should feel:
- editorial
- calm
- premium
- restrained

Avoid:
- futuristic typography
- playful rounded fonts
- loud display systems

Typography supports atmosphere.
It should not dominate the interface.

---

# Performance Rules

Performance is critical.

Prefer:
- SVG assets
- compressed imagery
- CSS layering
- subtle motion systems

Avoid:
- large animation libraries
- unnecessary rendering complexity
- GPU-heavy visual systems

The experience should feel premium and lightweight.

---

# Accessibility Rules

Always preserve:
- WCAG AA contrast
- keyboard navigation
- reduced motion support
- readable overlays
- focus visibility

Cinematic visuals must never compromise readability.

---

# Development Workflow

Claude should:
- work section-by-section
- avoid uncontrolled rewrites
- preserve existing atmosphere
- implement incrementally

Preferred workflow:
1. audit
2. prioritize
3. implement one section
4. review visually
5. refine

---

# Important Principle

Every implementation decision should reinforce:

“premium cinematic travel artifact rendered in software.”

Never:
“generic travel startup UI.”
