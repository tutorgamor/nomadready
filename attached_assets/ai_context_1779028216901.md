```md
# NomadReady — AI Context

## Product Identity
NomadReady is a premium cinematic editorial travel platform for digital nomads.

It must NOT feel like:
- SaaS dashboard
- travel booking app
- fintech UI
- startup landing page
- mobile app stretched to desktop

It MUST feel like:
- a beautifully crafted travel artifact rendered in software
- a cinematic field guide
- a luxury editorial travel publication
- a modern atlas
- a tactile passport system

Core emotional qualities:
- premium
- cinematic
- tactile
- atmospheric
- calm
- warm
- globally connected
- editorial
- desktop-native

## Visual North Star
“The Worn Atlas”

The product should evoke:
- Monocle
- Apple editorial product pages
- luxury travel magazines
- worn passports
- embossed paper documents
- field journals
- atlas textures

## Color Rules
Use warm neutrals only.

Primary palette:
- parchment
- ivory
- sand
- ember gold
- warm black

Avoid:
- cold gray
- blue tech gradients
- cyberpunk colors
- saturated neon

Gold/amber accents must stay restrained.

## Material Direction
Surfaces should feel tactile:
- paper grain
- leather texture
- embossed print
- atlas relief
- worn travel documents

Every card or panel should feel physical.

## Motion Rules
Motion should feel environmental.

Allowed:
- subtle atmospheric movement
- soft hover lift
- cinematic fades
- environmental depth

Avoid:
- bouncy animations
- startup microinteractions
- exaggerated motion
- playful scaling
- flashy transitions

## Layout Philosophy
Desktop layouts should feel editorial and spatial.

Avoid:
- centered SaaS layouts
- uniform card grids everywhere
- widget-heavy interfaces
- cramped compositions

Prefer:
- asymmetry
- negative space
- breathing room
- cinematic pacing
- layered composition
- immersive panels

## Homepage Structure
The homepage should contain:

1. Passport Gateway Hero
2. Featured Destination Panels
3. Secondary destination cards/grid
4. Editorial field notes sections
5. Atmospheric transitions

## Featured Destination Panels
Featured panels are large cinematic editorial sections.

They are NOT cards.

Each panel should:
- occupy large desktop space
- use immersive destination photography
- contain negative space for text
- feel like a magazine spread
- represent a travel emotion

Recommended featured destinations:
- Tokyo
- Indonesia/Bali
- Istanbul

## Destination Cards
Small destination cards still exist but are secondary.

Purpose:
- discovery
- navigation
- scanning destinations

They should not carry the entire emotional identity.

## Map Usage
The map is NOT a widget.

The map acts as:
- atmosphere
- environmental depth
- ambient world presence
- embossed atlas layer

No Google Maps feeling.
No markers.
No dashboard map UI.

## Passport Gateway
The passport object is a core brand moment.

Characteristics:
- tactile leather
- embossed gold details
- cinematic lighting
- subtle 2.5D tilt
- warm atmospheric shadows

Avoid:
- glossy 3D
- gaming aesthetics
- sci-fi visuals

## SVG / Editorial Assets
NomadReady uses:
- visa stamps
- route tickets
- postcards
- compass overlays
- atlas fragments
- field note graphics

These assets should feel integrated into the page material.

They are atmospheric graphic layers.
Not decorative stickers.

## Photography Style
Photography should feel:
- cinematic
- emotionally immersive
- quiet
- editorial
- tactile

Avoid:
- tourism brochure imagery
- influencers
- postcard landmarks
- cyberpunk visuals
- drone-shot energy

## Performance Rules
Keep effects lightweight.

Prefer:
- CSS gradients
- SVG overlays
- compressed MP4/WebM loops
- layered textures

Avoid:
- heavy WebGL
- complex 3D engines
- unnecessary animation systems

## Core Principle
NomadReady should feel like:

“a premium travel artifact rendered in software.”
```

# docs/implementation-rules.md

```md
# NomadReady — Implementation Rules

## General Philosophy
Implement interfaces as editorial scenes, not dashboards.

Every section should feel composed and intentional.

## Preferred Technologies
Use:
- Next.js
- Tailwind CSS
- CSS gradients
- SVG overlays
- lightweight motion
- compressed MP4/WebM loops when needed

Avoid:
- heavy WebGL
- unnecessary Three.js scenes
- overengineered animations

## Layout Rules
Desktop layouts should:
- use large visual sections
- preserve breathing room
- avoid excessive density
- create visual pacing

Avoid:
- tiny centered content columns
- overly symmetrical grids
- dashboard spacing systems

## Hero Rules
Hero sections should:
- feel cinematic
- contain atmospheric depth
- preserve text readability
- use directional lighting

Hero backgrounds should include calm zones for text overlays.

## Featured Panels
Featured destination panels:
- are large editorial sections
- should occupy meaningful vertical space
- should use layered imagery
- should contain integrated travel artifacts

Avoid generic card layouts.

## Destination Cards
Small cards should:
- remain secondary
- support discovery/navigation
- feel tactile and editorial

Avoid:
- dashboard widgets
- flat app cards
- generic SaaS styling

## Asset Rules
Editorial assets:
- use SVG whenever possible
- remain subtle
- integrate into compositions
- use low opacity

Assets include:
- stamps
- route tickets
- atlas marks
- postcards
- compass graphics

## Shadow Rules
Shadows must:
- be warm
- feel soft and layered
- avoid pure black

Preferred shadow behavior:
- gentle lift
- atmospheric diffusion
- restrained depth

## Motion Rules
Microinteractions:
- subtle
- restrained
- premium

Preferred durations:
- 150–260ms state changes
- slower cinematic fades when necessary

Avoid:
- bouncing
- exaggerated scaling
- noisy transitions

## Video Rules
Video should be used only as:
- ambient atmospheric loops
- environmental movement
- cinematic breathing

Avoid:
- autoplay-heavy experiences
- dramatic motion
- travel vlog aesthetics

## Accessibility
Always preserve:
- WCAG AA readability
- reduced motion support
- keyboard navigation
- sufficient contrast

## Performance
Keep the experience lightweight.

Prefer:
- compressed media
- SVG assets
- CSS layering
- subtle motion

Avoid:
- massive JS animation systems
- GPU-heavy effects
- unnecessary rendering complexity

## Development Workflow
Claude should:
- work section-by-section
- avoid large uncontrolled rewrites
- preserve the established visual language
- implement incrementally

## Important Principle
Every implementation decision should reinforce:

“premium cinematic travel artifact rendered in software.”
```
